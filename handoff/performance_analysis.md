# fastlckin 性能分析报告：与 lcMLkin、TKGWV2、READ 的综合对比

> 本文档分析 fastlckin v0.2.0 相比 lcMLkin v2.1 及其他同类工具的性能优化机制、数据结构设计、算法原理，并提供综合对比。

---

## 1. 概述

fastlckin 是对 lcMLkin v2.1（Python 实现）的完整 C++17 重写。核心目标是在**保留原版统计严谨性**（完整 GL 似然值建模 + Anderson & Weir IBS|IBD 概率模型 + FST 校正）的同时，通过消除 Python 解释器开销、外部 PLINK 调用和内存低效操作，实现显著的性能提升。

---

## 2. fastlckin 相比 lcMLkin v2.1 的性能优势分析

### 2.1 语言与运行时层

| 维度 | lcMLkin v2.1 | fastlckin | 性能影响 |
|------|-------------|-----------|---------|
| 语言 | Python 3 | C++17 | C++ 编译型代码比 Python 解释型快 10-100× |
| 依赖 | numpy, scipy | 无第三方 Python 库 | 避免 numpy/scipy 动态分发开销 |
| 并行 | `multiprocessing.Pool`（进程级） | `std::thread` + ThreadPool（线程级） | 线程共享内存，避免进程间序列化/反序列化 |

**关键差异**：Python 的 `multiprocessing` 使用进程间通信（IPC），每个进程需要拷贝完整数据（GL 矩阵 + IBS|IBD 矩阵）。fastlckin 的 `ThreadPool` 使用原生 C++ 线程，所有线程共享同一份 `_gl_matrix` 和 `_ibs_ibd` 矩阵，零拷贝。

### 2.2 最重大的优化：消除外部 PLINK 调用

**lcMLkin v2.1 的最大瓶颈**：每对样本需要调用 PLINK **两次**：
```
for each pair (i, j):
    subprocess.Popen(["plink", "--bfile", prefix, "--extract", snplist_file, ...])  # extract SNPs
    subprocess.Popen(["plink", "--bfile", prefix, "--extract", snplist_file, "--indep-pairwise", "50", "5", "0.8", ...])  # LD prune
    # 每次调用：启动 PLINK 进程 → 读取 .bed/.bim/.fam → 计算 → 写临时文件 → 解析输出
```

**快慢对比**：
- PLINK 每次启动 + I/O ≈ 0.1-0.5 秒
- 每对样本 2 次调用 → 0.2-1.0 秒/对
- 100 个样本 → 4950 对 → 约 16-82 分钟仅 LD 修剪

**fastlckin 的做法**：
- 一次性加载 `.bed` 到内存（或从 GL 计算期望基因型）
- C++ 内置滑窗 LD 修剪（`ld_prune()` / `ld_prune_from_likelihoods()`）
- 零外部进程、零临时文件、零 I/O
- LD 修剪时间从分钟级降至毫秒级

### 2.3 VCF 读取优化

| 操作 | lcMLkin v2.1 | fastlckin |
|------|-------------|-----------|
| 方式 | `file.readlines()` 一次性加载全部 VCF | `htslib` 流式逐条读取 `VCFRecord` |
| 内存 | O(全部VCF行数)，大VCF 可能 OOM | O(当前记录)，常量内存 |
| 过滤 | Python 循环逐行检查 | C++ 循环 + 直接跳过 |

**关键差异**：对于百万级 SNP 的 VCF，`readlines()` 可能消耗数 GB 内存。fastlckin 的流式读取仅需存储当前记录，内存占用稳定。

### 2.4 数据结构优化

| 操作 | lcMLkin v2.1 | fastlckin |
|------|-------------|-----------|
| 数据布局 | `[snp][sample]`（Python list） | `[sample][snp]`（转置后，C++ `vector`） |
| 缓存局部性 | pair 级计算时跨行访问 | pair 级计算时连续内存访问 |
| 预计算 | 无 | `_ibs_ibd[9][n_snps][3]` 全局预计算 + `pibs_per_snp` pair 级预计算 |

**关键优化**：VCF 天然按 `[snp][sample]` 组织，但 pair 级计算需要频繁访问同一 SNP 的两个样本。fastlckin 在加载时将数据转置为 `[sample][snp]`，使每个样本的 SNP 数据在连续内存中，大幅提升缓存命中率。

## 3. 数据结构构建过程

### 3.1 数据读取与转置

```
VCF/BCF 流式读取（htslib）
    ↓
临时缓冲: snp_gls[snp][sample]   // VCF 天然布局
    ↓
转置操作: _gl_matrix[sample][snp]  // pair 级计算友好
    ↓
每个元素: GenotypeLikelihood { gl[3], gq, masked }
```

**Mode 1（VCF-only）**：
- 保留所有双等位 SNP，无 `.bim` 过滤
- GL 来自 VCF 的 PL 或 GL 字段

**Mode 2（VCF+PLINK）**：
- 仅保留与 `.bim` 匹配的 SNP（`.bim` 作为白名单）
- GL 来自 VCF

**Mode 3（PLINK-only）**：
- 从 `.bed` 读取硬性基因型（0/1/2）
- 转换为 $\delta$ 函数似然值：`gl[g_obs] = 1.0, gl[其他] = 1e-10`

### 3.2 等位基因频率计算

| 模式 | 方法 | 公式/算法 |
|------|------|----------|
| Mode 1 | EM 算法（`compute_af_from_likelihoods`） | E-step: 后验 P(G\|D,p) ∝ GL × HWE 先验<br>M-step: p = Σ(P(G=1)+2P(G=2)) / 2N |
| Mode 2 | 从 `.bed` 计数或 `.frq` 读取 | 硬性基因型直接统计 |
| Mode 3 | 从 `.bed` 计数 | 硬性基因型直接统计 |

**EM 算法细节**：
```
初始化: p = 0.5
迭代:
  对每个样本 i:
    后验 ∝ GL_i × HWE(p)
    E[G_i] = P(G=1|D_i,p) + 2×P(G=2|D_i,p)
  p_new = Σ E[G_i] / (2N)
  直到 |p_new - p| < 1e-6
```

### 3.3 IBS|IBD 概率矩阵预计算

```
precompute_ibs_ibd_matrix()
    ↓
对每个 SNP s:
  对每个 IBD 状态 z ∈ {0, 1, 2}:
    对每个基因型组合 c ∈ {0..8}:
      分子 = Mij 乘积（Anderson & Weir 2007 序列抽取模型）
  归一化: B[c][s][z] ← B[c][s][z] / Σ_c' B[c'][s][z]
    ↓
结果: _ibs_ibd[9][n_snps][3]
```

**关键优化**：IBS|IBD 矩阵在所有样本对之间共享，只需计算一次。10 万个 SNP × 9 种组合 × 3 个 IBD 状态 = 270 万个 double，约 21 MB 内存，完全可预计算。

### 3.4 LD 修剪数据准备

| 模式 | 数据类型 | 计算方式 |
|------|---------|---------|
| Mode 1 | `_expected_genotypes[sample][snp]` | 后验 E[G] = P(G=1\|D,p) + 2×P(G=2\|D,p) |
| Mode 2/3 | `_bed_genotypes[sample][snp]` | 从 `.bed` 读取（0/1/2/-1） |

**跳过 LD 修剪**（`--no-ld-prune`）：
- `config.ld_config.skip = true`
- `_estimate_pair()` 直接使用所有未屏蔽的 SNP，不执行任何 LD 计算
- 适用于极低深度数据（古 DNA、NIPT），样本间共享 SNP 本就稀疏

### 3.5 数据矩阵总结

```txt
KinshipEstimator 内部数据矩阵:

_sample_names:         vector<string>                      // n_samples
_snp_infos:            vector<SNPInfo>                     // n_snps (chrom, pos, ref, alt, af, af_masked)
_gl_matrix:            vector<vector<GenotypeLikelihood>>  // [sample][snp]
_ibs_ibd:              vector<vector<array<double,3>>>     // [9][snp][3]
_bed_genotypes:        vector<vector<int8_t>>              // [sample][snp] (Mode 2/3 LD)
_expected_genotypes:   vector<vector<double>>              // [sample][snp] (Mode 1 LD)
_vcf_to_bim_index:     vector<int>                         // VCF SNP → .bim 索引映射 (Mode 2)
```

### 4.2 Stage 1 详解：数据转置——为 pair 级计算做缓存优化

**VCF 文件天然按 SNP 组织**，每行是一个位点上所有样本的基因型。但亲缘度估计的核心操作是**对每一对样本计算跨所有 SNP 的联合似然**，这要求频繁访问同一 SNP 上两个样本的数据（意思是对的，但下面的示意图是错的）。

```txt
VCF 天然布局: snp_gls[S][N]              转置后布局: _gl_matrix[N][S]
(N=4 样本, S=5 SNP)                      (N=4 样本, S=5 SNP)

         S0    S1    S2    S3    S4              S0    S1    S2    S3    S4
      ┌─────┬─────┬─────┬─────┬─────┐        ┌─────┬─────┬─────┬─────┬─────┐
 Ind0 │  a  │  b  │  c  │  d  │  e  │        │  a  │  f  │  k  │  p  │  u  │
      ├─────┼─────┼─────┼─────┼─────┤   ──▶  ├─────┼─────┼─────┼─────┼─────┤
 Ind1 │  f  │  g  │  h  │  i  │  j  │        │  b  │  g  │  l  │  q  │  v  │
      ├─────┼─────┼─────┼─────┼─────┤        ├─────┼─────┼─────┼─────┼─────┤
 Ind2 │  k  │  l  │  m  │  n  │  o  │        │  c  │  h  │  m  │  r  │  w  │
      ├─────┼─────┼─────┼─────┼─────┤        ├─────┼─────┼─────┼─────┼─────┤
 Ind3 │  p  │  q  │  r  │  s  │  t  │        │  d  │  i  │  n  │  s  │  x  │
      └─────┴─────┴─────┴─────┴─────┘        │  e  │  j  │  o  │  t  │  y  │
                                             └─────┴─────┴─────┴─────┴─────┘

 每一行 = 一个 SNP 的全部样本 GL         每一行 = 一个样本的全部 SNP GL
 读取 Ind0 和 Ind1 的 SNP2 数据:         读取 Ind0 和 Ind1 的 SNP2 数据:
   → 需跨行跳跃到 row 2 取 (c, h)         → Ind0 行内直接取 gl[2]，Ind1 行取 gl[2]
   → 缓存不友好！                           → 连续内存访问，缓存友好！
```

**为什么这很重要？** 现代 CPU 的 L1 缓存约 32-64 KB，一次缓存行加载 64 字节（8 个 double）。转置后，样本 i 的连续 SNP 数据在内存中紧密排列，CPU 预取器可以高效预测访问模式，缓存命中率从约 1/8 提升到接近 100%。

**数据结构选择**：
```cpp
// 每个 SNP 的似然值用定长数组存储，确保 24 字节对齐
struct GenotypeLikelihood {
    double gl[3];    // P(G=0|D), P(G=1|D), P(G=2|D)  — 固定 24 字节
    double gq;       // 基因型质量分数
    bool masked;     // 是否屏蔽（低质量或缺失）
};
// 内存中: gl[0], gl[1], gl[2], gq 连续排列 → 一次 SIMD 可加载全部 4 个 double
```

### 4.3 Stage 2 详解：EM 频率估计——利用 GL 数据结构的迭代精炼

EM 算法的核心洞察：**每个 SNP 的频率估计只依赖该 SNP 列上所有样本的 GL 值**，因此可以对每个 SNP 独立迭代。

```
对单个 SNP s 的 EM 迭代过程:

  ┌─────────────────────────────────────────────────────┐
  │  初始化: p = 0.5 (非信息先验)                          │
  └───────────────────────┬─────────────────────────────┘
                          │
                          ▼
  ┌─────────────────────────────────────────────────────┐
  │  E-step: 对每个样本 i，计算后验基因型概率                │
  │                                                     │
  │  ┌──────────────────────────────────────────────┐   │
  │  │  prior[0] = (1-p)²     ← HWE: P(G=0|p)       │   │
  │  │  prior[1] = 2p(1-p)    ← HWE: P(G=1|p)       │   │
  │  │  prior[2] = p²         ← HWE: P(G=2|p)       │   │
  │  │                                              │   │
  │  │  post[g] = gl_i[g] × prior[g]  (贝叶斯更新)    │   │
  │  │  post_norm = post / Σ post                   │   │
  │  │                                              │   │
  │  │  E[G_i] = post_norm[1] + 2 × post_norm[2]    │   │
  │  └──────────────────────────────────────────────┘   │
  │                                                     │
  │  这里 gl_i[0..2] 直接来自 _gl_matrix[i][s].gl[]       │
  │  → 连续内存读取，无随机访问                             │
  └───────────────────────┬─────────────────────────────┘
                          │
                          ▼
  ┌─────────────────────────────────────────────────────┐
  │  M-step: 更新频率                                    │
  │                                                     │
  │  p_new = Σ E[G_i] / (2 × N_valid)                   │
  │                                                     │
  │  分子 = 所有样本的期望等位基因剂量之和                    │
  │  分母 = 2 × 有效样本数                                │
  └───────────────────────┬─────────────────────────────┘
                          │
                          ▼
  ┌─────────────────────────────────────────────────────┐
  │  收敛判定: |p_new - p| < 1e-6 ?                      │
  │    Yes → 输出 af = p_new                             │
  │    No  → p = p_new, 回到 E-step                      │
  └─────────────────────────────────────────────────────┘
```

**准确性保证**：
- EM 算法在 HWE 先验下单调递增对数似然，保证收敛到局部最优
- 使用 `clamp(1e-10, 1-1e-10)` 防止频率收敛到边界 0 或 1（避免后续 log(0) 问题）
- 每个 SNP 独立估计，不存在 SNP 间的交叉污染

**效率保证**：
- 时间复杂度：$O(S \times N \times I)$，其中 $I$ 为 EM 迭代次数（通常 5-20 次即收敛）
- 对每个 SNP，仅遍历 `_gl_matrix[*][s].gl[]`——转置后的列访问模式
- 无外部依赖，无临时文件，无进程启动

### 4.4 Stage 3a 详解：IBS|IBD 预计算——一次计算，所有 pair 共享

IBS|IBD 矩阵是整个算法中最关键的共享数据结构。其核心数学原理来自 Anderson & Weir (2007) 的序列抽取模型。

```
对单个 SNP（频率 p, FST=F）的 IBS|IBD 概率计算:

  ┌────────────────────────────────────────────────────────────────┐
  │  Mij 基础函数: M(p, F, i) = (1-F)·p + i·F                       │
  │                                                                │
  │  物理含义: 从一个携带 i 个目标等位基因的池子中                        │
  │  不放回抽取一个等位基因，抽到目标等位基因的概率                        │
  │                                                                │
  │  F=0 时: M(p,0,i) = p  (随机交配，无结构)                         │
  │  F=1 时: M(p,1,i) = i  (完全近交，固定)                           │
  └────────────────────────────────────────────────────────────────┘

  IBS\IBD     z=0 (无关)           z=1 (一个共享)         z=2 (两个共享)
  ┌─────────┬──────────────────┬──────────────────┬──────────────────┐
  │ PPQQ    │ M(p,1)M(p,0)     │ 0                │ 0                │
  │         │ × M(q,1)M(q,0)   │                  │                  │
  │         │ (4次独立抽取)      │                  │                  │
  ├─────────┼──────────────────┼──────────────────┼──────────────────┤
  │ PPPQ    │ 2·M(p,2)M(p,1)   │ M(p,1)M(p,0)     │ 0                │
  │         │   M(p,0)M(q,0)   │   M(q,0)         │                  │
  │         │ (4次抽取)         │ (3次抽取)         │                  │
  ├─────────┼──────────────────┼──────────────────┼──────────────────┤
  │ PPPP    │ M(p,3)M(p,2)     │ M(p,2)M(p,1)     │ M(p,1)M(p,0)     │
  │         │   M(p,1)M(p,0)   │   M(p,0)         │                  │
  │         │ (4次抽取)         │ (3次抽取)         │ (2次抽取)         │
  ├─────────┼──────────────────┼──────────────────┼──────────────────┤
  │ PQPQ    │ 4·M(p,1)M(p,0)   │ 2·[M(p,1)M(q,0)² │ 2·M(p,0)M(q,1)   │
  │         │   M(q,1)M(q,0)   │  + M(q,1)M(p,1)  │                  │
  │         │                  │    M(p,0)]       │                  │
  └─────────┴──────────────────┴──────────────────┴──────────────────┘

  ┌────────────────────────────────────────────────────────────────┐
  │  动态归一化: 对每个 z，令 B[c][z] /= Σ_c' B[c'][z]                │
  │                                                                │
  │  这保证: Σ_c B[c][z] = 1  对所有 z, p, F 精确成立                 │
  │                                                                │
  │  为什么需要? 因为 IBD=1 时抽取次数为 3（不是 4），                    │
  │  总概率质量不等于 IBD=0 时的 4 次抽取概率质量。                      │
  │  归一化使不同 z 的概率在可比较的尺度上。                             │
  └────────────────────────────────────────────────────────────────┘
```

**预计算的效率优势**：

```
不预计算（lcMLkin 方式）:             预计算（fastlckin 方式）:

对每对 (i,j):                        全局一次性计算:
  对每个 SNP s:                        对每个 SNP s:
    计算 B[9][3]  ← Mij 公式             计算 B[9][3]  ← Mij 公式
    → 9×3 = 27 次 Mij 调用               → 存入 _ibs_ibd[9][s][3]
  → 重复 N_pairs 次！
                                     对每对 (i,j):
总计算量:                                直接从 _ibs_ibd[c][s][z] 读取
  S × 27 × N_pairs                      → 27 次数组查找（无计算）

S=100K, N_pairs=4950:               总计算量:
  ≈ 13.2 亿次 Mij 调用                S × 27 = 270 万次 Mij 调用
                                     + N_pairs × S × 27 次查找

                                     加速比: ~4950× (Mij 调用次数)
```

### 4.5 Stage 3b 详解：LD 修剪（LD prune）数据结构——滑窗 + 贪心移除

LD 修剪（LD prune）的目标是从高度相关的 SNP 中选出独立子集，减少冗余信息对似然函数的影响。

```
LD 修剪的滑窗贪心过程 (window=50, step=25, r²_threshold=0.5):

  候选 SNP 列表 (unmasked):
  ┌──┬──┬──┬──┬──┬──┬──┬──┬──┬──┬─-─┬─-─┬─-─┬─-─┬─-─┬──┐
  │S0│S1│S2│S3│S4│S5│S6│S7│S8│S9│S10│S11│S12│S13│S14│...
  └──┴──┴──┴──┴──┴──┴──┴──┴──┴──┴──-┴───┴───┴───┴───┴───

  Window 1: [S0, S1, ..., S49]  (前 50 个候选 SNP)
  ┌────────────────────────────────────────────────┐
  │  计算 active 中所有 SNP 对的 r²                  │
  │                                                │
  │  r²(S3, S7) = 0.82  > 0.5  ✗ 高 LD             │
  │  r²(S3, S12) = 0.71 > 0.5  ✗ 高 LD             │
  │  r²(S3, S25) = 0.65 > 0.5  ✗ 高 LD             │
  │  r²(S7, S12) = 0.55 > 0.5  ✗ 高 LD             │
  │  ...                                           │
  │                                                │
  │  统计每个 SNP 的高 LD 伙伴数:                     │
  │  S3: 3 个伙伴 (S7, S12, S25)  ← 最多！           │
  │  S7: 2 个伙伴 (S3, S12)                         │
  │  S12: 2 个伙伴 (S3, S7)                         │
  │                                                │
  │  → 移除 S3（与最多 SNP 高 LD）                    │
  │  → 重新检查直到无高 LD 对                         │
  └────────────────────────────────────────────────┘

  Window 2: [S25, S26, ..., S74]  (步进 25 个 SNP)
  ┌────────────────────────────────────────────────┐
  │  仅检查新进入窗口的 SNP 与 active 的 LD            │
  │  已在上一个窗口保留的 SNP 不会被重新移除             │
  └────────────────────────────────────────────────┘

  最终 retained[]:
  ┌──┬──┬──┬──┬──┬──┬──┬──┬─-─┬─-─┬──┐
  │S0│S1│S2│S4│S5│S6│S8│S9│S10│S11│...   ← LD 独立的 SNP 子集
  └──┴──┴──┴──┴──┴──┴──┴──┴──-┴-──┴──┴──
```

**两种 r² 计算模式的数学差异**：

```txt
Mode 2/3 — 硬性基因型 Pearson r²:

  r²(S_a, S_b) = [Σ(g_a·g_b)/n - μ_a·μ_b]² / (σ²_a · σ²_b)

  其中 g ∈ {0, 1, 2}，是离散的整数值
  → 直接读取 _bed_genotypes[sample][snp] (int8_t，1字节/元素)

Mode 1 — 期望基因型 Pearson r²:

  r²(S_a, S_b) = [Σ(E[G_a]·E[G_b])/n - μ_a·μ_b]² / (σ²_a · σ²_b)

  其中 E[G] = P(G=1|D,p) + 2·P(G=2|D,p) ∈ [0, 2]，是连续值
  → 读取 _expected_genotypes[sample][snp] (double，8字节/元素)

  期望基因型 E[G] 的计算利用了 HWE 后验:
    E[G] = (gl[1]·2pq + 2·gl[2]·p²) / (gl[0]·q² + gl[1]·2pq + gl[2]·p²)
  → 这自然地融合了测序不确定性：低质量数据的 E[G] 趋向先验均值 2p
```

### 4.6 Stage 4 详解：Pair 级估计——PIBS 预计算与目标函数求值

这是整个流水线中计算量最大的阶段，也是对数据结构设计最敏感的环节。

```
对样本对 (ind1, ind2) 的估计过程:

  Step ① 合并 mask:
  ┌──────────────────────────────────────────────────────────────┐
  │  mask[s] = af_masked[s]                                      │
  │           ∨ _gl_matrix[ind1][s].masked                       │
  │           ∨ _gl_matrix[ind2][s].masked                       │
  │                                                              │
  │  → 三种来源中任一不可用的 SNP 被排除                              │
  └──────────────────────────────────────────────────────────────┘

  Step ② LD 修剪 → retained[] (见 Stage 3b)

  Step ③ 预计算 PIBS (Pair-wise IBS):
  ┌──────────────────────────────────────────────────────────────┐
  │  对每个 retained SNP ri:                                      │
  │                                                              │
  │  gl1 = _gl_matrix[ind1][retained[ri]]  ← 连续内存，无跳跃       │
  │  gl2 = _gl_matrix[ind2][retained[ri]]  ← 连续内存，无跳跃       │
  │                                                              │
  │  pibs[ri][PPQQ] = gl1.gl[0] × gl2.gl[2]   (ind1=RR,ind2=AA)  │
  │  pibs[ri][QQPP] = gl1.gl[2] × gl2.gl[0]   (ind1=AA,ind2=RR)  │
  │  pibs[ri][PPPQ] = gl1.gl[0] × gl2.gl[1]   (ind1=RR,ind2=RA)  │
  │  pibs[ri][PQPP] = gl1.gl[1] × gl2.gl[0]   (ind1=RA,ind2=RR)  │
  │  pibs[ri][PQQQ] = gl1.gl[1] × gl2.gl[2]   (ind1=RA,ind2=AA)  │
  │  pibs[ri][QQPQ] = gl1.gl[2] × gl2.gl[1]   (ind1=AA,ind2=RA)  │
  │  pibs[ri][PPPP] = gl1.gl[0] × gl2.gl[0]   (ind1=RR,ind2=RR)  │
  │  pibs[ri][PQPQ] = gl1.gl[1] × gl2.gl[1]   (ind1=RA,ind2=RA)  │
  │  pibs[ri][QQQQ] = gl1.gl[2] × gl2.gl[2]   (ind1=AA,ind2=AA)  │
  │                                                              │
  │  → 9 次乘法/retained SNP，结果存入栈上 std::array<double,9>     │
  └──────────────────────────────────────────────────────────────┘

  Step ④ Nelder-Mead 优化目标函数求值:
  ┌─────────────────────────────────────────────────────────----─────┐
  │  每次目标函数求值（被 Nelder-Mead 调用 ~100-1000 次）:                │
  │                                                                  │
  │  log_L = 0                                                       │
  │  for ri in retained:                                             │
  │    site_prob = 0                                                 │
  │    for c in 0..8:                                                │
  │      ibd_sum = k₀·B[c][ri][0] + k₁·B[c][ri][1] + k₂·B[c][ri][2]  │
  │                     ↑                                            │
  │                     └── 从预计算的 _ibs_ibd 矩阵直接读取             │
  │                         3 次乘法 + 2 次加法                        │
  │                                                                  │
  │      site_prob += pibs[ri][c] × ibd_sum                          │
  │                     ↑                                            │
  │                     └── 从预计算的 pibs 数组直接读取                 │
  │                         1 次乘法 + 1 次加法                        │
  │                                                                  │
  │    log_L += ln(site_prob)                                        │
  │                                                                  │
  │  return -log_L  (取负号，因为 Nelder-Mead 做最小化)                 │
  └──────────────────────────────────────────────────────────----────┘
```

**每次目标函数求值的运算量**：

| 操作  | 次数/retained SNP  |  总次数（S_retained 个 SNP） |
|------|--------------------|----------------------------|
| 乘法 | 3 (ibd_sum) + 1 (pibs) = 4 | 4 × S_retained |
| 加法 | 2 (ibd_sum) + 1 (site_prob) = 3 | 3 × S_retained |
| log | 1 | S_retained |
| **合计** | **8 次浮点运算** | **~8 × S_retained** |

若 retained 有 5000 个 SNP，每次目标函数求值仅 ~40000 次浮点运算。Nelder-Mead 约需 200-500 次求值 → **每次 pair 估计约 800 万-2000 万次浮点运算**，在现代 CPU 上仅需约 2-5 毫秒。

### 4.7 数据流与算法协同设计的核心思想

fastlckin 的数据结构和算法设计遵循三个协同原则：

```
原则 1: 数据布局服务于访问模式
─────────────────────────────
  访问模式: pair 级 → 需要 (ind1, s) 和 (ind2, s) 的数据
  数据布局: [sample][snp] → 每个样本的 SNP 数据连续
  结果: 缓存命中率 ≈ 100%（而非 [snp][sample] 的 ≈ 12.5%）

原则 2: 共享计算全局预计算，pair 特有计算 pair 级预计算
─────────────────────────────────────────────────────────
  IBS|IBD 矩阵: 仅依赖 AF + FST，与 pair 无关
    → 全局预计算一次，O(S) 时间，所有 N_pairs 对共享
  PIBS: 依赖 (ind1, ind2) 的 GL 乘积，每对不同
    → pair 级预计算，在 Nelder-Mead 调用前一次性完成
  目标函数求值: 仅做查表 + 乘加
    → 被 Nelder-Mead 调用数百次时零冗余计算

原则 3: 数学精确性与计算效率不矛盾
─────────────────────────────────────
  Mij 动态归一化: 数学上保证 ΣB=1，同时预计算后仅增加
                   9 次除法/SNP/IBD 状态（可忽略不计）
  EM 频率估计: 使用 GL 而非硬性基因型，每次迭代多 3 次乘法/样本，
               但消除了分型错误带来的系统性偏差
  Cotterman 约束: 1 次乘法 + 1 次比较/目标函数求值（O(1) 开销），
                  防止物理不可行解
```

**三层预计算的时间复杂度对比**：

```
                    不预计算          仅全局预计算        两层预计算 (fastlckin)
                    ─────────         ──────────        ──────────────────
IBS|IBD 计算:      O(S·N_pairs)      O(S)              O(S)
PIBS 计算:         O(S·N_pairs·I)    O(S·N_pairs·I)    O(S_ret·N_pairs)
目标函数求值:      O(S·I·R)          O(S·I·R)          O(S_ret·I·R)
─────────────────────────────────────────────────────────────────────────
总浮点运算量       ~10¹²             ~10¹¹              ~10⁹
(S=100K, N=100,    (每对重新计算       (IBS|IBD 共享       (PIBS 预缓存 +
 I=500 次求值)      Mij 公式)          PIBS 仍每对算)       Nelder-Mead 查表)

→ 三层预计算策略将总运算量降低约 1000 倍
```

## 5. 核心算法原理

### 5.1 GLkin 最大似然估计（MLE）

**目标函数**（负对数似然，需最小化）：

$$\mathcal{L}(k_1, k_2) = -\sum_{s} \ln \left( \sum_{c=0}^{8} \text{PIBS}[s, c] \cdot \sum_{z=0}^{2} B[c][s][z] \cdot k_z \right)$$

其中：
- $k_0 = 1 - k_1 - k_2$（IBD=0 概率）
- $k_1$（IBD=1 概率）
- $k_2$（IBD=2 概率）
- $\text{PIBS}[s, c] = L_{i,G_1(c)}[s] \times L_{j,G_2(c)}[s]$（两个样本的联合似然值）
- $B[c][s][z] = P(\text{IBS}=c \mid \text{IBD}=z)$（Anderson & Weir 条件概率）

**约束**：
- $k_0, k_1, k_2 \geq 0$，$k_0 + k_1 + k_2 = 1$
- $4 \cdot k_0 \cdot k_2 \leq k_1^2$（Cotterman 不等式）

### 5.2 Nelder-Mead 单纯形法

- 2D 参数空间优化（$k_1, k_2$），$k_0$ 自动推导
- 多重启策略：默认 3 次
  - 第 1 次：$(k_1, k_2) = (0, 0)$（确定性起点）
  - 其余：`random_ibd_start()`（满足约束的随机起点）
- 收敛标准：$x_{tol} = 0.01$，$f_{tol} = 0.01$

### 5.3 滑窗 LD 修剪算法

```
输入: 候选 SNP 列表 candidates[]
输出: 保留的 SNP 列表 retained[]

for start = 0, step, 2*step, ...:
  end = min(start + window_size, n_candidates)
  active = candidates[start:end] 中未被排除的 SNP
  
  while 存在高 LD 对:
    对 active 中每对 (s1, s2):
      计算 r²(s1, s2)
    找出与最多其他 SNP 高 LD 的 SNP
    将其从 active 中移除
    
retained = 所有未被排除的 candidate
```

**r² 计算**：
- 硬性基因型（Mode 2/3）：Pearson r² on {0, 1, 2}
- 期望基因型（Mode 1）：Pearson r² on 连续值 E[G] ∈ [0, 2]

### 5.4 Mij 序列抽取模型（IBS|IBD 概率）

核心公式：$M(p, F, i) = (1 - F) \cdot p + i \cdot F$

- $p$：等位基因频率
- $F$：FST 值（群体分化程度）
- $i$：已抽取等位基因中目标等位基因的计数

**IBD=0**（两个无关个体，4 次独立抽取）：
- $B[\text{ppqq}][0] = M(p,1) \cdot M(p,0) \cdot M(q,1) \cdot M(q,0)$

**IBD=1**（一个共享等位基因，3 次抽取）：
- $B[\text{pppq}][1] = M(p,1) \cdot M(p,0) \cdot M(q,0)$

**IBD=2**（两个等位基因均共享，2 次抽取）：
- $B[\text{pppp}][2] = M(p,1) \cdot M(p,0)$

**动态归一化**（v0.2.0 关键修复）：
- 对每个 IBD 状态 $z$，计算 $\sum_c B[c][z]$，然后 $B[c][z] \leftarrow B[c][z] / \sum_c B[c][z]$
- 保证 $\sum_c B[c][z] = 1$ 对所有 $z$、所有 $p$、所有 $F$ 精确成立

## 7. fastlckin 的架构优势总结

### 7.1 设计哲学

fastlckin 的核心设计哲学是：**在 C++ 中完整实现 lcMLkin 的统计算法，同时消除所有非计算瓶颈**。

具体做法：
1. **消除外部依赖**：LD 修剪从 PLINK 调用改为 C++ 内置
2. **优化内存使用**：VCF 流式读取，数据转置，预计算共享矩阵
3. **原生并行**：C++ 线程池替代 Python 多进程
4. **统一框架**：所有模式统一到 GL 似然值框架（PLINK-only 也转为 δ 函数）

### 7.2 关键性能数据（估算）

| 场景 | lcMLkin v2.1 | fastlckin | 加速比 |
|------|-------------|-----------|--------|
| 50 样本，10 万 SNP | ~2 小时 | ~1-5 分钟 | **20-120×** |
| 100 样本，50 万 SNP | ~20 小时 | ~10-30 分钟 | **40-120×** |
| 200 样本，100 万 SNP | ~数天 | ~1-2 小时 | **>100×** |

> 注：具体加速比取决于 PLINK 启动时间、磁盘 I/O 速度、线程数等因素。

### 7.3 v0.2.0 新增功能

- **三种输入模式**：VCF-only（古 DNA / 无参考面板）、VCF+PLINK（参考面板）、PLINK-only（芯片数据）
- **EM 频率估计**：无 PLINK 时从 GL 推算等位基因频率
- **期望基因型 LD 修剪**：基于后验 E[G] 的 Pearson r² 计算
- **跳过 LD 修剪**（`--no-ld-prune`）：极低深度数据优化
- **Mij 动态归一化**：修复 IBD=1 下的归一化错误
- **静态构建**（`STATIC_BUILD`）：支持 Linux/macOS 静态链接分发
- **完整测试套件**：10 个测试文件，全面覆盖核心算法

## 8. 结论

fastlckin 通过以下方式实现了比 lcMLkin v2.1 显著的性能提升：

1. **消除 PLINK 外部调用**：这是最大的优化点，将每对样本的 LD 修剪时间从秒级降至毫秒级。
2. **C++ vs Python**：编译型语言 + 原生线程池 + 零拷贝共享内存。
3. **流式 VCF 读取**：常量内存占用，支持大规模 VCF。
4. **数据转置**：`[sample][snp]` 布局优化 pair 级计算的缓存局部性。
5. **预计算**：IBS|IBD 矩阵全局预计算，PIBS pair 级预计算。
6. **统一框架**：所有输入模式统一到 GL 似然值框架，代码简洁且易于维护。

与 TKGWV2 和 READ 相比，fastlckin 在**统计严谨性**（完整 GL 建模 + Mij 动态归一化）和**计算性能**（C++ 线程池）上均具有显著优势，同时通过三种输入模式保持了足够的灵活性，适用于从古代 DNA 到标准芯片数据的多种场景。

## 9. 统计学严谨性深度分析：优势与潜在缺陷

> 本章从统计理论、优化方法和边界条件出发，系统评估 fastlckin v0.2.0 的统计严谨性，指出已知的优势和潜在缺陷，为后续版本优化提供决策依据。

### 9.1 统计学优势

#### 9.1.1 完整基因型似然值（GL）建模

fastlckin 的核心优势在于**不在似然值空间做任何简化假设**。三种输入模式最终都统一到 `gl[3]` 框架：

- **Mode 1（VCF-only）**：直接从 VCF 的 PL/GL 字段提取，完整保留测序不确定性
- **Mode 2（VCF+PLINK）**：同上，PLINK 仅用于 LD 修剪的硬性基因型
- **Mode 3（PLINK-only）**：转为 δ 函数 `gl[g_obs]=1.0, gl[其他]=1e-10`

与 TKGWV2 的伪单倍体采样（每个位点随机抽一个等位基因）相比，GL 建模保留了三种基因型的完整后验分布，在低覆盖度下尤其重要。

#### 9.1.2 Mij 动态归一化（v0.2.0 关键修复）

***[2026-06-23]这里还是需要反复确认***

Anderson & Weir (2007) 的序列抽取模型通过 `M(p, F, i) = (1-F)·p + i·F` 精确描述 FST 校正下的等位基因抽取概率。v0.2.0 引入的**动态归一化**（见 `ibd_model.cpp` L93-103）保证：

$$\sum_{c=0}^{8} B[c][z] = 1 \quad \forall z \in \{0, 1, 2\}, \forall p \in (0,1), \forall F \in [0,1]$$

这在 lcMLkin v2.1 中使用固定分母 D₀/D₁/D₂ 时并不成立，导致 IBD=1 下的概率归一化存在系统性偏差。fastlckin 的修复在数学上是严格的。

#### 9.1.3 Cotterman 不等式约束

目标函数中实施了：
$$4 k_0 k_2 \leq k_1^2$$

这是从孟德尔遗传的 Cotterman 矩阵推导出的必要条件，防止优化器收敛到物理上不可行的 IBD 组合。lcMLkin v2.1 和 TKGWV2/READ 均不实施此约束。

#### 9.1.4 EM 算法频率估计

Mode 1 中从 GL 矩阵通过 EM 算法估计等位基因频率，使用 Hardy-Weinberg 先验：
- E-step：$P(G_i | D_i, p) \propto L(D_i | G_i) \cdot P(G_i | p)$
- M-step：$p^{(t+1)} = \frac{\sum_i E[G_i]}{2N}$

这比 PLINK 的硬性基因型计数更准确地反映低覆盖度下的等位基因频率分布。

#### 9.1.5 逐 SNP 预计算 PIBS

`pibs_per_snp[ri][c] = gl1[c₁] × gl2[c₂]` 在每个 retained SNP 上独立计算，不做 SNP 间的平均或近似。这保证了似然函数对每个 SNP 的贡献是精确的。

### 9.2 潜在缺陷与局限

#### 9.2.1 Nelder-Mead 优化器的局限性（中等优先级）

**问题描述**：当前使用 Nelder-Mead 单纯形法（无梯度方法）优化 2D 目标函数，存在以下局限：

| 局限 | 影响 |
|------|------|
| **收敛精度有限** | `xtol=0.01, ftol=0.01` 意味着 k₁/k₂ 的估计精度约 ±0.01，对区分 2 级 vs 3 级亲属可能不够 |
| **局部最优风险** | 仅 3 次重启（默认），若似然面有多个局部极值可能遗漏全局最优 |
| **约束处理粗糙** | Cotterman 不等式通过返回 `1e10` 惩罚，导致单纯形在边界附近行为不佳 |
| **无梯度信息** | 无法计算标准误、置信区间 |

**建议优化（v0.3.0）**：
- 引入 **L-BFGS-B** 或 **投影梯度法**，支持盒约束 + Cotterman 不等式
- 利用 GLkin 似然函数的解析梯度（可推导闭式表达）
- 增大重启次数至 5-10 次，或使用全局优化（如 basin-hopping）
- 收紧收敛阈值至 `1e-4` 或 `1e-5`

#### 9.2.2 EM 频率估计的偏差（高优先级）

**问题描述**：

1. **留一偏差（Leave-one-out bias）**：EM 使用所有样本（包括待估计的对）来估计频率，然后在 GLkin 似然中使用同一频率。这导致频率估计与亲缘度估计之间存在耦合——对于近亲对，其共同的等位基因会被过度计入频率，进而影响 IBD 估计。

2. **无 FST 校正**：EM 先验使用 $P(G|p) = \text{HWE}(p)$，未考虑 FST。在结构化群体中，HWE 先验是有偏的，真实先验应为 $P(G|p,F)$。

3. **收敛初值敏感**：所有 SNP 统一从 `p=0.5` 初始化。对于极低频变异（MAF < 0.01），EM 可能收敛到错误的局部极值。

**建议优化（v0.3.0）**：
- 对每对样本使用**留一频率**（排除 ind1, ind2 后重新估计），或使用外部参考面板频率
- EM 先验中引入 FST 校正：`prior[0] = M(q,1)*M(q,0)`, `prior[1] = 2*M(p,0)*M(q,0)`, `prior[2] = M(p,1)*M(p,0)`
- 允许用户指定初始频率或使用 PLINK 频率作为 EM 初值

#### 9.2.3 无置信区间或标准误（高优先级）

**问题描述**：当前输出仅为点估计 `(k₀, k₁, k₂)` 和 `log_likelihood`，缺少任何不确定性度量。

| 对比 | fastlckin | READ | lcMLkin |
|------|-----------|------|----------|
| 点估计 | ✓ | ✓ | ✓ |
| 对数似然 | ✓ | ✗ | ✗ |
| 标准误 | ✗ | ✗ | ✗ |
| 置信区间 | ✗ | ✗ | ✗ |
| Z-score | ✗ | ✓ | ✗ |

**建议优化（v0.3.0）**：
- **Fisher 信息矩阵**：在 MLE 点计算 Hessian 的逆，得到渐近标准误
- **Profile likelihood CI**：固定 k₁ 或 k₂，profile 出另一个参数的 CI
- **Bootstrap**：对 retained SNPs 进行自助抽样，得到经验分布
- **似然比检验**：输出 MLE 与无关系假设（k₀=1, k₁=k₂=0）的对数似然比

#### 9.2.4 δ 函数似然值的任意性（低优先级）

**问题描述**：Mode 3 将硬性基因型转为 δ 函数似然值：
```cpp
gl[g_obs] = 1.0;
gl[其他] = 1e-10;   // EPS，任意选择
```

- `1e-10` 是一个经验选择，不是统计推导。虽然不影响 MLE 的最优点（因为 MLE 只依赖于 `gl` 的比值），但在数值上可能影响 Nelder-Mead 的收敛行为。
- 更严格的做法是**在 Mode 3 下简化似然函数**，直接将 IBS 状态作为观测值，跳过 GL 中间层。

**建议优化（v0.4.0）**：
- 为 Mode 3 实现专用似然函数（直接使用 IBS 观测值）
- 或验证 `1e-10` 不影响结果（敏感性分析）

#### 9.2.5 LD 修剪的效率与正确性（中等优先级）

**问题描述**：

1. **per-pair 修剪**：每对样本独立执行 LD 修剪（`ld_prune()` 在 `_estimate_pair()` 内调用），不同对使用不同的 retained SNP 集合。这在统计上是合理的（排除对特定的 masked SNP），但效率较低。

2. **`get_snp_column()` 内存分配**：每次调用从 `[sample][snp]` 布局中提取一列 SNP 到新的 `vector`，在窗口循环内反复调用，导致大量临时分配。

3. **贪心移除策略**：每次迭代移除与最多 SNP 高 LD 的 SNP，是 PLINK 风格的启发式方法，不一定得到最优独立集。

**建议优化（v0.3.0）**：
- 提供 **全局 LD 修剪选项**：对所有样本使用统一的 unmasked SNP 集合进行一次性 LD 修剪，大幅减少计算量
- 将 `get_snp_column()` 改为就地计算（避免 vector 拷贝）
- 使用 `std::vector<bool>` 或 bitset 替代 `unordered_set<int>` 存储 excluded

#### 9.2.6 单一 FST 参数（低优先级）

**问题描述**：`compute_ibs_ibd_probs(af, fst)` 对所有 SNP 使用相同的 FST 值。真实数据中 FST 随基因组位置变化（选择位点、结构变异区域可能有更高的局部分化）。

**建议优化（v0.4.0）**：
- 支持**逐 SNP FST 向量**（从外部文件读取）
- 或使用 Weir & Goudet (2017) 的方法估计全局 FST

#### 9.2.7 样本量可扩展性 O(n²)（中等优先级）

**问题描述**：所有样本对的 IBD 估计复杂度为 $O(n^2 \cdot S \cdot R)$，其中 $n$ 为样本数、$S$ 为 SNP 数、$R$ 为 Nelder-Mead 重启次数。对于 1000 个样本（约 50 万对），即使使用多线程，计算量仍然巨大。

**建议优化（v0.4.0+）**：
- **分层策略**：先用快速方法（如 KING-robust 估计 PI_HAT）筛查，仅对 PI_HAT > 阈值的对运行完整 GLkin
- **子集估计**：提供 `--pairs` 参数，仅估计用户指定的样本对
- **批量优化**：将 PIBS 和 IBS|IBD 的访问模式改为向量化友好的布局（SoA → AoS）

#### 9.2.8 关系分类阈值硬编码（低优先级）

**问题描述**：`classify_relationship()` 使用固定的 PI_HAT 阈值：
```
> 0.708 → Duplicate/MZ
0.354-0.708 → Parent-Offspring / Full-Sibling
0.177-0.354 → Second-degree
0.0884-0.177 → Third-degree
< 0.0884 → Unrelated
```

这些阈值来自理论值的中点，但：
- 低覆盖度下 PI_HAT 估计值有系统性偏差，固定阈值可能导致错误分类
- 无法区分 4 级及以上亲属
- 无概率分类（如 "95% 概率为二级亲属"）

**建议优化（v0.4.0）**：
- 支持用户自定义阈值
- 引入基于似然比或后验概率的分类框架
- 对低覆盖度数据提供分类置信度度量

### 9.3 缺陷优先级矩阵

| 优先级 | 缺陷 | 影响 | 修复复杂度 | 建议版本 |
|--------|------|------|------------|----------|
| **P0** | EM 频率估计偏差 | 系统性影响 k₀/k₁/k₂ 估计精度 | 中等 | v0.3.0 |
| **P0** | 无置信区间/标准误 | 无法评估估计可靠性 | 中等 | v0.3.0 |
| **P1** | Nelder-Mead 收敛精度不足 | 边界附近估计不准 | 高（需替换优化器） | v0.3.0 |
| **P1** | per-pair LD 修剪效率 | 大规模样本性能瓶颈 | 低 | v0.3.0 |
| **P1** | O(n²) 可扩展性 | 千样本级别计算量过大 | 高（需引入筛查策略） | v0.4.0+ |
| **P2** | δ 函数似然值任意性 | Mode 3 数值稳定性 | 低 | v0.4.0 |
| **P2** | 单一 FST | 结构化群体精度下降 | 中等 | v0.4.0 |
| **P2** | 分类阈值硬编码 | 低覆盖度分类不准 | 低 | v0.4.0 |

### 9.4 与 lcMLkin v2.1 统计学严谨性的具体差异

| 维度 | fastlckin v0.2.0 | lcMLkin v2.1 | 差异分析 |
|------|-----------------|-------------|----------|
| **IBD 概率模型** | Mij + 动态归一化 ✓ | 固定 D₀/D₁/D₂ ✗ | lcMLkin 在 IBD=1 下的概率归一化有系统性错误 |
| **Cotterman 约束** | 4k₀k₂ ≤ k₁² ✓ | 无 ✗ | fastlckin 防止不可行解 |
| **频率来源** | EM / .bed / .frq | 仅 .bed / .frq | fastlckin 在无 PLINK 时仍可估计 |
| **PL 归一化不变性** | 已验证 ✓ | 未考虑 | PL min-subtraction 不影响 GLkin |
| **多重启策略** | 确定性 + 随机 ✓ | 仅确定性 | fastlckin 减少局部最优风险 |
| **约束处理** | 惩罚函数（1e10） | 无约束 | 两者均可改进为更优雅的约束优化 |

### 9.5 总结：fastlckin 的统计学定位

fastlckin v0.2.0 在**统计模型层面**（GL 似然值 + Mij 动态归一化 + Cotterman 约束）已达到同类工具中的最高水平。其主要改进空间在**数值优化层面**（优化器精度、约束处理）和**推断层面**（置信区间、频率偏差校正），这些属于统计软件从"功能正确"到"生产级可靠"的标准进化路径。

**建议的优化路线**：
1. **v0.3.0**（统计可靠性）：EM 留一频率 + Fisher 信息矩阵 CI + 改进优化器
2. **v0.4.0**（功能扩展）：全局 LD 修剪 + 分层策略 + 概率分类 + FST 向量
3. **v1.0.0**（生产就绪）：完整 benchmark 套件 + 大规模验证 + 论文发表

#### 阶段 2：v0.2.0 vs v0.3.0 回归对比

**目标**: 量化 v0.3.0 改进带来的实际效果

**对比指标**:

1. **估计精度**
   ```
   对于已知关系样本对:
     error_v020 = |k_est - k_true|
     error_v030 = |k_est - k_true|
     improvement = (error_v020 - error_v030) / error_v020 × 100%
   ```

2. **频率估计偏差**
   ```
   比较全量 EM 和留一法 EM 的频率差异:
     delta_af = |af_full - af_loo|
     预期: 近亲对的 delta_af 应显著大于无关对
   ```

3. **标准误有效性**
   ```
   验证标准误是否反映真实不确定性:
     - 高质量数据 → SE 小
     - 低质量数据 → SE 大
     - 95% 置信区间应覆盖 95% 的真实值
   ```

**对比实验设计**:

```python
import pandas as pd
import numpy as np

# 运行两个版本
df_v020 = pd.read_csv("output_v020.tsv", sep="\t", comment="#")
df_v030 = pd.read_csv("output_v030.tsv", sep="\t", comment="#")

# 对比已知亲子对
po_pairs = df_v020[df_v020['relationship'] == 'Parent-Offspring']

# 计算改进
k1_error_v020 = np.abs(po_pairs['k1'] - 1.0).mean()
k1_error_v030 = np.abs(po_pairs['k1_v030'] - 1.0).mean()

print(f"k1 估计精度改进: {(k1_error_v020 - k1_error_v030) / k1_error_v020 * 100:.1f}%")

# 对比标准误
print(f"v0.3.0 平均 SE(k1): {df_v030['se_k1'].mean():.4f}")
print(f"v0.3.0 平均 SE(k2): {df_v030['se_k2'].mean():.4f}")
```
